# Baseline A11y Tests

## What We Test
- Keyboard navigation (ArrowDown, ArrowUp, Home, End) with focus tracking and `aria-activedescendant`
- Selection via Space, Enter, and click with `selection:change` event emission
- Group header skipping during keyboard navigation (items with `__groupHeader` flag)
- Interactive vs non-interactive mode (`role=listbox` vs `role=list`, tabindex presence)
- `aria-activedescendant` cleanup on focusout (inside vs outside the vlist root)

## Test Groups
- **keyboard navigation** (6 tests): ArrowDown/Up moves focus and emits `focus:change`, Home/End jump to first/last, ArrowLeft/Right are no-ops on vertical lists, boundary clamping at last item
- **selection** (6 tests): Space and Enter toggle selection on focused item, click selects/deselects items, Space is a no-op when no item is focused, aria-selected toggling via click with forceRender
- **group header skipping** (2 tests): ArrowDown skips `__groupHeader` items, group headers cannot be selected with Space
- **interactive: false** (4 tests): content gets `role=list` instead of `listbox`, no tabindex, keyboard events do not move focus, default interactive mode uses `listbox`
- **aria-activedescendant cleanup** (2 tests): focusout to outside element removes `aria-activedescendant`, focusout to element inside vlist root preserves it

## Known Gaps
- New in v2 (no v1 equivalent) -- no regression comparison available
- No tests for horizontal keyboard navigation (ArrowLeft/Right in horizontal mode)
- No tests for multi-select keyboard patterns (Shift+Arrow, Ctrl+Space)
