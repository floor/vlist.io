# A11y

Baseline keyboard navigation, single-select, and focus management. Opt-in plugin that adds ARIA-compliant interaction without the full `selection()` plugin.

```ts
import { createVList, a11y } from "vlist";

const list = createVList({
  container: "#app",
  item: { height: 48, template: renderItem },
  items: data,
}, [a11y()]);

list.on("focus:change", ({ id, index }) => {
  console.log("Focused:", id);
});
```

### When to use

| Scenario | Plugin |
|----------|--------|
| No keyboard nav needed | Neither |
| Basic keyboard nav + single-select | `a11y()` |
| Multi-select, range, Ctrl+A | `selection()` |

The `a11y()` plugin is a no-op when `selection()` is active — you don't need both.

### Keyboard

| Key | Action |
|-----|--------|
| Arrow Up / Down | Move focus by 1 (list), by columns (grid), within lane (masonry) |
| Arrow Left / Right | Disabled in list mode. Move by 1 in grid, adjacent lane in masonry |
| Home / End | Jump to first / last item |
| Page Up / Down | Jump by visible page |
| Space / Enter | Toggle selection on focused item |

Navigation adapts automatically when combined with `grid()` or `masonry()` — no extra configuration needed.

### Events

| Event | Payload |
|-------|---------|
| `focus:change` | `{ id: string \| number, index: number }` |
| `selection:change` | `{ selected: (string \| number)[], items: T[] }` |

### ARIA

Sets `aria-activedescendant` on the content element to track keyboard focus. Items receive focus and selection state via the item state callback, which the render pipeline uses for `aria-selected` and CSS classes (`{prefix}-item--focused`, `{prefix}-item--selected`).

Requires `interactive: true` (the default) in the list config for `role="listbox"` and `tabindex="0"` on the content element.

### Notes

- Group headers (`__groupHeader: true`) are automatically skipped during navigation.
- Click selects the clicked item; keyboard focus ring is not shown on click.
- Focus is restored to the last focused item when the list regains focus.
- Priority 55 — runs after `selection()` (50) and layout plugins like `grid()` (10) and `masonry()` (10).

See [Accessibility Guide](../accessibility.md) for the full ARIA structure and screen reader recommendations.
