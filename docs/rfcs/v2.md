# RFC-002: vlist v2 Definitive Master Specification

## The 2-Phase Pipeline (Strict Zero-Allocation)

The 3-phase pipeline concept is dead. This is a strict 2-phase synchronous pipeline, supported by an external asynchronous invalidation loop.

### Phase 1: Calculate & Reconcile (Zero-Allocation Hot Path)
The `EngineState` is a persistent singleton instantiated **once** during `.build()`. It maintains pre-allocated Typed Arrays. The pipeline never allocates intermediate `{ node, transformY, itemData }` objects.

```typescript
class EngineState {
  // Persisted pool. Mutated in-place by Phase 1. Read directly by Phase 2.
  public readonly visibleIndices = new Int32Array(500); 
  public readonly visibleOffsets = new Float64Array(500);
  public readonly visibleSizes = new Float64Array(500);
  public visibleCount = 0;
}
```
Phase 1 yields a `LayoutWindow` structure. The v1 empty range sentinel (`start: 0, end: -1`) is explicitly encoded as `count = 0` and `startIndex = 0`.

### Phase 2: Commit (Direct Buffer Reads)
Phase 2 reads directly from the `EngineState` buffers. It does not allocate arrays or intermediate objects. It iterates `for (let i = 0; i < state.visibleCount; i++)` and directly applies `transformY = state.visibleOffsets[i]` to the DOM nodes. This perfectly mirrors the performance profile of `src/builder/core.ts:731-943` but actively enforces zero-allocation.

## External Asynchronous Invalidation (ResizeObserver)
`ResizeObserver` is NOT a pipeline phase. It is an asynchronous observer that manages the size cache.
When a measurement differs from the cache, it updates the cache and schedules a new Phase 1 run via `requestAnimationFrame`. This solves the measurement feedback loop without polluting the 2-Phase synchronous pipeline model.

## Build-Time Compiled Hooks
No closures are created on the hot path. Hooks are collected into an array during the cold-path `.build()` and iterated linearly:
```typescript
for (let i = 0; i < calculateHooks.length; i++) {
  calculateHooks[i](singletonState); // Mutates persistent state in-place
}
```

## Strict v1 Compliance Guarantees
1. **Render Count Safety Cap:** Explicitly preserved exactly as `src/builder/core.ts:765-778`.
2. **Zero Container Size Early Exit:** Explicitly preserved exactly as `src/builder/core.ts:745-751`.
3. **Empty Range Sentinel:** Encoded natively via `visibleCount = 0` on the `LayoutWindow`.
4. **Overscan Application:** Explicitly preserved exactly as `src/builder/range.ts:44-50`.
5. **Compressed Mode:** Phase 1 maps virtual offsets to physical indices exactly as `v1:784-790`.
6. **Synchronous Wheel Rendering:** Wheel events trigger `engine.render()` synchronously, bypassing `rAF` exactly as `src/builder/core.ts:1030-1085`.
